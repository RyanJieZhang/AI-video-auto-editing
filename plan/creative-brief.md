# Creative Brief

## Objective

制作一个 60 秒竖屏说明视频，讲清楚“为什么不要用一个万能 prompt 自动剪视频”，并展示本项目的分层 Agent 工作流为什么更稳定。

## Audience

- B 站、小红书上的 AI 工具使用者
- 想用 AI 做视频但经常卡在素材、脚本、剪辑、导出混乱的人
- 对 Codex、Remotion、自动化剪辑 workflow 感兴趣的创作者

## Platform & Format

- 平台：B 站竖屏、小红书
- 画幅：9:16
- 分辨率：1080 x 1920
- 时长：60 秒
- 语言：中文

## Tone

清晰、节奏快、偏教学型，但有一点锋利的观点感。

## Core Message

自动剪视频不能靠一个万能 prompt。更稳定的方法是把视频创作拆成多个职责清晰的 Agent 层：规划、脚本、素材、对齐、动效、音频、Remotion、QA、发布和复盘。

## Hook

“如果你让一个 prompt 同时写脚本、找素材、剪视频、配音乐、写代码，它很快就会乱。”

## Chapter Structure

1. 问题：万能 prompt 会把任务揉成一团。
2. 后果：脚本、素材、时间线、音乐、代码互相污染。
3. 解法：拆成职责清晰的 10 层 Agent。
4. 示例：每一层只交付一个明确产物。
5. 收束：第一次慢一点，之后每次都更快。

## Visual Direction

- 竖屏信息流风格
- 高对比卡片
- 蓝色代表系统/结构，橙色代表风险/警告
- 以文字动效、流程卡、对比卡、进度条为主
- 不使用外部版权素材，先用 Remotion 生成图形素材

## Constraints

- 第一版用 Remotion 生成图形和字幕，不依赖外部视频素材。
- 先做可预览的 MVP，不追求最终商业质感。
- 口播暂不生成真实 TTS 音频，先用字幕和音频计划表达。

## Open Questions

- 后续是否需要真实 AI 配音？
- 是否需要加入屏幕录制或真实项目界面素材？
- 是否需要为 B 站和小红书分别做封面文案？
